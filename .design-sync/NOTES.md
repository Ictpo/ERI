# ERI design-sync notes

## Repo shape
- Package shape (no Storybook). Components live in `frontend/components/ui/*.tsx`.
- `frontend/package.json` name is `frontend`; bundle global is `window.ERI`.
- No dist build for the UI kit — bundle is built from a hand-written entry
  `frontend/.ds-entry.ts` (`export *` of each ui/*.tsx). Pass it via `--entry`.
- CSS: ERI has no shipped stylesheet; the compiled Tailwind CSS lives in the
  Next static export at `frontend/out/_next/static/css/*.css`. `buildCmd` runs
  `npm run build` then `.design-sync/prep-css.mjs`, which: strips the next/font
  `@font-face` rules into `.design-sync/assets/inter.css` (rewriting the
  `/_next/...` urls to local woff2 copies) AND adds a `body { font-family }`
  rule so designs get Inter (the app applies it via a next/font body class the
  bundle never sees). Main CSS minus those rules -> `frontend/.ds-styles.css`
  (cfg.cssEntry). extraFonts points at inter.css.

## Component API gotchas (IMPORTANT for authoring previews)
- ERI's components are NOT full Radix/shadcn compounds. Several export only a
  single element with NO sub-components:
  - `Alert` — single div; content via children (use a `<div fontWeight:600>`
    title + a `<div>` body). There is NO AlertTitle/AlertDescription.
  - `Badge`, `Button`, `Input`, `Textarea`, `Label`, `Skeleton`, `Switch`,
    `Slider` — single elements.
  - `Card` DOES export CardHeader/CardTitle/CardDescription/CardContent/CardFooter.
  - `Dialog`, `Select`, `Tabs`, `Tooltip`, `Toast` are Radix-based compounds
    (DialogContent, SelectTrigger/Content/Item, TabsList/Trigger/Content, etc.).
  ALWAYS grep the component's `export {` line before authoring — do not assume
  shadcn sub-component names exist.
- Real Button variants: default, secondary, outline, ghost, destructive, link.
  Sizes: default, sm, lg, icon. Primary color is indigo-600.
- Alert variants: default, warning, destructive, info.
- Badge variants: default, secondary, outline, success, warning (+ more — grep).

## Overlay/interaction components
- Dialog/Select open states and Tooltip need cardMode:"single" + a viewport,
  or they collapse. Author them composed with a trigger; set overrides as needed.

## Re-sync risks
- The bundle entry `frontend/.ds-entry.ts` is hand-maintained: if a ui/*.tsx is
  added/removed, update it AND cfg.componentSrcMap.
- prep-css depends on the Next static export existing (`npm run build` first)
  and on next/font hashed class names (`__Inter_*`); a Next/font upgrade can
  change those — re-check the body font stack if text renders as Times New Roman.
- Fonts are the app's next/font woff2 copies; no license text shipped.

## Toast — ships the floor card (deliberate)
- Radix Toast renders live (probe: 76px, opacity 1) but will not screenshot in
  isolated `?story=` capture — the provider/portal + enter animation collapse
  the measured output to blank. Two rework attempts (position:static, no
  ToastViewport, relative wrapper) didn't fix the static capture. Toast is fully
  importable/functional from window.ERI; it just ships the honest floor card.
  A future authoring attempt would need a non-animated static Toast composition.

## Committed sync inputs (do not delete)
- `frontend/.ds-entry.ts` — hand-written bundle entry (`export *` of ui/*.tsx).
  Committed; re-sync passes it via `--entry frontend/.ds-entry.ts`.
- `.design-sync/prep-css.mjs` — CSS/font prep, run by buildCmd after `npm run build`.
- `.design-sync/assets/` is gitignored (regenerated by prep-css from the Next build).
- Re-sync command (from repo root):
  `node .ds-sync/resync.mjs --config .design-sync/config.json --node-modules frontend/node_modules --entry frontend/.ds-entry.ts --out ./ds-bundle --remote .design-sync/.cache/remote-sync.json`
  (fetch `_ds_sync.json` from the project into that remote path first).
